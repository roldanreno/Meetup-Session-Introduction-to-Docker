Docker? I know the whale but what is it for?

Docker is a tool designed to make it easier to create, deploy, and run applications by using containers that allow developers packing applications with all of the parts needed to run, such as libraries and other dependencies, and ship it all out as one package.

Docker images are the basis of containers

An Image is a collection of root filesystem changes and the corresponding execution parameters for use within a container runtime stacked on top of each other. An image does not have state and it never changes.

Ok then... so Docker is?

Docker is a containerization platform where we can package our application with its libraries and dependencies inside the container. Docker Container is a somewhat like a virtual machine.
Unlike a virtual machine, Docker container shares the same Linux kernel as the system.

Benefits

  • Versatile
  • Lightweight
  • Efficient
  • Isolated

Let's do something more complex

$ docker run --hostname=quickstart.cloudera
--name=clouderareno --privileged=true -ti 
-p 8890:8888 -p 7180:7180 -p 10080:80 
-v /home/reno/Documents/CDHDockerFolder:/src 
--publish-all=true -d cloudera/quickstart 
/usr/bin/docker-quickstart
  • By using docker run --privileged, container can not only access to all hosts devices but also use most of host computer’s kernel functions
  • -i means interactive mode
  • -t means open it in terminal
  • -v allows to share volume
  • -p port redirection
  • --publish-all=true opens up all the host ports to the docker ports, so you can access programs like the Hue in the port 8888 and YARN in the port 8088
  • -d runs container in background and print container ID
$ docker ps -a
$ docker rm -f ID

Last but not least how to build a Dockerfile!

A Docker File is a simple text file with instructions on how to build your images.

FROM ubuntu:latest
MAINTAINER roldanreno@gmail.com
RUN apt-get update -y
RUN apt-get install -y python-pip python-dev build-essential
COPY . /app
WORKDIR /app
RUN pip install -r requirements.txt
ENTRYPOINT["python"]
CMD ["app.py"]
  • RUN executes command(s) in a new layer and creates a new image.
  • CMD sets default command and/or parameters, which can be overwritten from command line when docker container runs.
  • ENTRYPOINT configures a container that will run as an executable.
